================================================================================
  StudentCabinet — личный кабинет ВятГУ (краткое описание)
================================================================================

СТЕК
  Flask + PostgreSQL (БД StudentCabinet), сессии в cookie.
  Статика: CSS/JS, PWA (manifest, service worker).
  Внешнее: VK (новости), Google Maps (карта/маршруты), Яндекс (геокод корпусов).

РОЛИ
  student   — студент
  teacher   — преподаватель
  admin     — администратор
  guest     — без входа (только общие страницы)

--------------------------------------------------------------------------------
ОБЩИЕ ЭКРАНЫ (все / гость)
--------------------------------------------------------------------------------

  /  Главная
     Лендинг: вход, ссылки на разделы, блок «возможности».

  /login  Вход
     Email + пароль. После входа — редирект по роли.

  /register  Регистрация
     Отключена (сообщение «аккаунты создаёт админ»).

  /news  Новости
     Лента из кэша БД (парсинг vk.com/kollegevyatsu, без токена; до 10 постов).

  /map  Карта и контакты
     Список корпусов/общежитий, виджет Яндекс.Карт (без JS API), маршрут —
     ссылка на yandex.ru/maps. Координаты: data/campus_coords.json,
     data/campus_overrides.json; синхронизация: parsers/vyatsu_campus.py.

  /faq  Вопросы и ответы
     Справка из БД.

--------------------------------------------------------------------------------
СТУДЕНТ
--------------------------------------------------------------------------------

  /dashboard  Кабинет
     Карточки-ссылки: расписание, студбилет, запись к преподавателю, FAQ.

  /schedule  Расписание
     Календарь (неделя/день/месяц): пары группы из Excel + слоты записи.
     Клик по событию — модалка: заметка, вложения (если есть).

  /student_card  Цифровой студбилет
     Данные и фото из профиля (заполняет админ).

  /appointment  Запись к преподавателю
     Список преподавателей → выбор.

  /appointment/<id>  Окна приёма преподавателя
     Слоты по датам, кнопка «Записаться», предупреждение при пересечении с парой.

  /appointment/mine  → редирект на /appointment

--------------------------------------------------------------------------------
ПРЕПОДАВАТЕЛЬ
--------------------------------------------------------------------------------

  /teacher/dashboard  Кабинет
     Слева — список своих слотов приёма; справа — записавшиеся студенты,
     кнопка «Отказать». Ссылка в расписание для создания слотов.

  /teacher/schedule  Расписание
     Календарь: пары колледжа (Excel), пары ВятГУ (парсер), слоты приёма.
     Клик по пустой ячейке — модалка «Новое окно приёма» (время, кабинет,
     тема, аудитория, группы). Клик по событию — редактирование/удаление слота,
     заметки, файлы для студентов.

--------------------------------------------------------------------------------
АДМИНИСТРАТОР  (/admin)
--------------------------------------------------------------------------------

  Вкладка «Студенты»
     Создание, фильтр по группе, поиск, пагинация, правка/удаление.

  Вкладка «Преподаватели»
     Создание, привязка к ФИО из Excel, кабинет, статус синхронизации ВятГУ.

  Вкладка «Расписание»
     Загрузка Excel (.xlsx), дата «действует с».

  Вкладка «Справка VK»
     Новости VK (парсинг группы); кнопка синхронизации корпусов с vyatsu.ru.

--------------------------------------------------------------------------------
МОДАЛЬНЫЕ ОКНА (не отдельные URL)
--------------------------------------------------------------------------------

  Модалка события календаря
     Заметка пользователя, вложения, для препода — правка слота приёма.

  Модалка «Новое окно приёма» (расписание преподавателя)
     Создание слота после клика по ячейке календаря.

--------------------------------------------------------------------------------
API (JSON, для фронта)
--------------------------------------------------------------------------------

  /api/schedule/events          — события календаря (по роли)
  /api/calendar/event-meta      — метаданные события
  /api/calendar/notes           — сохранить заметку
  /api/calendar/attachments     — загрузить файл к событию
  /api/teacher/slots            — список/создание слотов
  /api/teacher/slots/<id>       — изменить/удалить слот
  /api/teacher/bookings/<id>    — статус записи (отказ и т.д.)
  /api/groups/search            — поиск групп

--------------------------------------------------------------------------------
КАРТА КАМПУСА (таблица buildings) — 3НФ
--------------------------------------------------------------------------------

  Одна таблица buildings для корпусов и общежитий (~31 объект).
  Поле kind: building | dorm; уникальность (number, kind) — «корпус №5»
  и «общежитие №5» разные строки. Атрибуты (адрес, телефон, lat/lon,
  contact_person, contact_role, extra_info) зависят от id здания, не от
  kind транзитивно — схема соответствует 3НФ.

  Сравнение с users + roles:
    roles (справочник)     →  опционально campus_place_types (не внедрено)
    users.role_id          →  сейчас buildings.kind (CHECK)
    student_profiles       →  отдельная 1:1 «детали» не нужна: поля общие

  classrooms.building_id → buildings.id; запросы: db/queries/buildings.py.
  Парсеры: parsers/vyatsu_buildings.py, parsers/vyatsu_dorms.py.
  Рефакторинг (type_id без details) — только при необходимости новых
  типов объектов или единообразия с roles; для 3НФ не обязателен.

--------------------------------------------------------------------------------
ДАННЫЕ И СКРИПТЫ (вне веб-UI)
--------------------------------------------------------------------------------

  database/init_db.py           — создание БД и админа
  parsers/vyatsu_campus.py      — корпуса/общежития → БД
  parsers/vk_news_sync.py         — новости VK → кэш БД
  parsers/vk_schedule_sync.py   — расписание из VK
  schedule_parser/              — разбор Excel расписания
  data/campus_coords.json       — координаты для карты
